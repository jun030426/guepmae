# 웹 프로그래밍 기말 프로젝트 기획서

---

## 1. 세부 주제명

**방치된 공공데이터의 가치 재생산 — 실거래가 기반 부동산 급매 검증 플랫폼**

대주제 "재생산 또는 재활용"을 **공공데이터의 정보 자산 재생산**으로 재해석하였다. 정부가 공개하지만 일반 사용자에게는 가공되지 않은 CSV 형태로 방치되어 있는 국토교통부 실거래가 데이터를, 매물의 진짜 시세를 검증할 수 있는 의사결정 정보로 다층 재생산하는 것을 핵심 과제로 삼는다.

---

## 2. 개발 배경 (300자 내외)

부동산 플랫폼에서 "급매"라는 표시는 흔하지만, 정말 시세보다 싼지 검증할 방법이 없다. 국토교통부가 매월 공개하는 실거래가 데이터(공공데이터)는 자유롭게 활용 가능하지만, 일반 사용자에게는 17개 시도별로 분리된 가공되지 않은 CSV 파일 형태로 사실상 **방치**되어 있다. 흘러가면 사라지는 유휴 자원처럼, 활용되지 않는 공공데이터를 사용자가 직접 판단에 쓸 수 있는 정보로 **재생산**하는 것이 디지털 자원의 재활용이라 판단하였다. 이를 위해 전국 1년치 약 55만 건의 거래 데이터를 집계하여 모든 매물의 할인율을 자동 검증하고, 동일한 가공 결과를 차트·카드·AI 분석 여러 시각화로 다중 활용하는 웹 플랫폼을 React 기반으로 개발하였다.

## 3. 주요 기능

| 번호 | 기능 | 설명 |
|---|---|---|
| 1 | **매물 자동 검증 시스템** | 등록 시점에 같은 단지·평형의 1년 거래 중앙값(peer median)을 자동 산출해 할인율 계산. 5% 이상이면 "급매"로 분류 |
| 2 | **매물 목록 + 다중 필터** (`/properties`) | 지역(13시도)·가격대·면적대·할인율 필터, URL 쿼리 동기화, 페이지네이션, 3종 정렬 |
| 3 | **매물 상세 페이지** (`/properties/:id`) | 사진 갤러리(풀스크린) + 가격 검증 차트(13개월 추이) + AI 매물 리포트 5섹션 + 지도 + 360 파노라마 |
| 4 | **지도 검색** (`/map`) | Google Maps 전국 매물 핀 + 마커 클러스터링, 할인율 등급별 핀 색상 |
| 5 | **급매 리포트** (`/report`) | 시장 통계 8개 영역 인터랙티브 시각화 (헤드라인 카드 4 + 분류 카드 + 4종 차트 + Top 10 단지 + AI 시장 분석) |
| 6 | **AI 매물 리포트 자동 생성** | 매물 등록 시 백그라운드로 Google Gemini가 5섹션 자동 작성 (가격·매물·위치·사진·종합) |
| 7 | **AI 시장 분석** | 17개 시도 통계를 Gemini가 3~5개 인사이트 카드로 자동 진단 (예측·권유 금지 강제) |
| 8 | **중개사 portal** (`/agent/*`) | 가입 신청 → 승인 → 매물 등록 → 본인 매물 관리 |
| 9 | **사용자 인증 시스템** | Supabase Auth — 이메일+6자리 OTP / Google·Kakao OAuth / 비밀번호 재설정 3단계 / 5단계 역할 시스템 |
| 10 | **운영자 admin** (`/admin`) | 매물 검증 토글, 사용자 역할 관리, 중개사 신청 승인/거절 |

## 6. 재생산·재활용 구현 ★

본 프로젝트는 "재활용"을 단일 층위가 아닌 **3중 재활용 구조**로 구현하였다.

### 6-1. 1차 재활용 — 방치된 공공데이터를 정보 자산으로 재생산

```
[원시 자원]                       [재생산 결과]
국토부 CSV (CP949 인코딩)    →    구조화된 통계 + DB 적재
17개 시도, 108MB, raw           17개 시도 평균/중앙값,
                                 13개월 시계열, Top 10 단지,
                                 단지·평형 중앙값 41,141그룹
```

**처리 파이프라인** (`scripts/import-trades-csv.mjs` 등 Node.js 스크립트 5개):
1. `iconv-lite`로 CP949 → UTF-8 디코딩
2. `csv-parse`로 17개 CSV 일괄 파싱 → 약 55만 6,907건 정규화
3. **peer median 알고리즘**: 같은 단지+같은 면적대의 1년 중앙값을 기준가로 사용 (표본 3건 이상일 때만)
4. 거래마다 할인율 계산: `(peer median − 거래가) / peer median × 100`
5. 5개 차원으로 집계: 시도별·월별·면적대별·단지별·전체 인사이트
6. 결과를 JSON으로 출력 → Supabase `market_snapshots` 테이블에 7개 key/value로 적재

**재활용 가치 측정**:
- 원시 CSV: 누구나 볼 수 있지만 일반 사용자에게는 무의미한 raw 데이터
- 재생산 결과: 매물 한 건마다 "peer 대비 -8.3% 싸게 거래되어 급매 기준 충족" 같은 즉시 판단 가능한 정보

### 6-2. 2차 재활용 — 동일 데이터의 다중 시각화 재활용

한 번 가공된 `market_snapshots` 데이터(약 9.9KB)를 **8개의 다른 시각화 영역**에서 재활용한다. 각 영역은 동일한 데이터를 다른 관점·다른 컴포넌트로 보여준다.

| 시각화 영역 | 사용한 데이터 키 | 어떤 관점 |
|---|---|---|
| 상단 헤드라인 카드 4 (`HeadlineCard`) | monthly + regional + insights | "급매 압력" / "할인 강도" / "주목 지역" / "거래 모멘텀" |
| 프리미엄/할인 분류 카드 (`MarketCategoryCards`) | regional 17개 | 평균 음수/양수로 양분 분류 |
| 지역별 시세 대비 등락 (Recharts `ComposedChart`) | regional | 평균(막대) + 중앙값(점) 동시 표시 |
| 월별 추이 (Recharts `LineChart`) | monthly | 거래량 + 급매 매물 추이, 부분월 점선 처리 |
| 면적대 분포 (Recharts `BarChart`) | area_type | 중앙값 + 할인율 + 표본 3축 |
| 급매 레이더 (`RegionRadar`, 4탭) | regional | 거래량/급매비율/평균할인/중앙값 탭 전환 |
| Top 10 단지 표 | top_urgent | 표 형태 |
| AI 시장 분석 (`AiMarketReport`) | 7개 키 전체 | Gemini가 자연어로 진단 |

→ **한 번의 가공 → 8가지 의사결정 관점**. 사용자는 동일한 1년치 데이터를 자기 관심사에 맞는 각도에서 본다.

### 6-3. 3차 재활용 — React 컴포넌트의 재사용

기말 주제 "재활용"을 코드 차원에서도 일관되게 구현. **실제 import grep 기준 13개 재사용 패턴**.

## 7. 차별점 / 창의성 포인트

### 7-1. "주장"이 아닌 "증명" — 데이터 기반 자동 검증
기존 부동산 플랫폼은 매물이 "급매!"로 표기되더라도 사용자가 직접 시세를 검증해야 한다. 본 프로젝트는 매물 등록 시점에 peer median 알고리즘으로 할인율을 자동 계산하여, **모든 매물에 "검증된 할인율" 배지**가 자동으로 따라붙는다.

### 7-2. Outlier 분리 — 평균 vs 중앙값 동시 표시
데이터 분석 결과 서울 평균 할인율 `-32.8%`(매우 큰 음수)는 강남 신축 펜트하우스 등 소수 고가 거래(outlier)가 평균을 왜곡한 것이며, 일반 매물의 중앙값은 `-1.1%`로 거의 시세대로 거래되고 있음을 발견했다. 평균만 표시할 경우 사용자가 "서울 33% 폭락?"으로 오해할 위험이 있어, **평균과 중앙값을 동시에 표시하고 차이가 큰 지역은 자동 해설을 생성**하는 정직한 시각화를 구현하였다.

### 7-3. AI 안전 가드 — 법적 위험을 인지한 설계
부동산 정보는 자본시장법 위험이 있어 AI 생성 콘텐츠에 4중 안전망 적용:
| 보호 장치 | 구현 |
|---|---|
| 출력 형식 강제 | Zod schema + `generateObject` API |
| 예측 표현 금지 | 시스템 프롬프트로 "오를"·"전망"·"예상" 차단 |
| 권유 표현 금지 | "사세요"·"매수 적기"·"추천" 차단 |
| 면책 라벨 | LLM 출력이 아닌 UI 정적 텍스트로 일관 |

### 7-4. 데이터 정직성의 자체 검증
모든 통계의 정직성을 일회성 검증 스크립트로 입증:
- `verify-report-snapshot.mjs` — Top 10 단지가 진짜 CSV에 있는지 단지명·표본 수 직접 추적
- `verify-region-discount.mjs` — 17개 시도 평균/중앙값 차이 정량화
- `verify-price-trends.mjs` — 1,107 그룹의 13개월 충실도 분석

발표 시 "데이터가 진짜냐?" 질문에 검증 스크립트 실행으로 즉시 증명 가능.

### 7-5. End-to-End 흐름의 완결성
중개사 매물 등록 → 자동 할인율 검증 → AI 리포트 자동 생성 → 매수자가 시장 통계와 매물 시세 모두 확인 → 매수 결정. 단순한 정보 표시 사이트가 아닌 **데이터 파이프라인이 비즈니스 흐름과 일치하는 완결형 플랫폼**.
